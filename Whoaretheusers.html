There is some of this kind of automation in AOSP based custom firmwares, LOS, CrDroid, Pixel Experience etc, but for nowadays none of them has done this setting in so adjustable, flexible and comprehensive way. I believe that within 6 months beta-testers and other users will give many and more main use cases, so we can focus on them to improve functionality further in the way users like to.
